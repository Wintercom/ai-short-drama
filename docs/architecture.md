# 架构设计文档

> AI 短剧创作智能体 —— 多智能体协同的"文本→视频"工业化创作系统

## 1. 设计目标

| 目标层级 | 内容 | 落地情况 |
|---------|------|---------|
| 基础目标 | 跑通"文本→视频"完整闭环，流程不中断 | ✅ 零配置 6 秒产出 18s 成片 |
| 进阶目标 | 解决 AI 视频行业痛点，体现工程化与架构能力 | ✅ 见第 5 节 |

## 2. 五层架构总览

```
┌──────────────────────────────────────────────────────────┐
│  接入层  cmd/drama          CLI 入口，组装并启动流水线       │
├──────────────────────────────────────────────────────────┤
│  ① 总控调度层  orchestrator                                │
│     Pipeline(DAG) · Runner(并发/续跑) · Checkpoint · State │
├──────────────────────────────────────────────────────────┤
│  ② 智能体层  agents                                        │
│   剧本引擎 │ 资产/角色管理 │ 视觉分镜 │ 音频合成 │ 后期合成   │
├──────────────────────────────────────────────────────────┤
│  ③ 能力服务层  services（接口 + 可插拔实现）                │
│     LLM · T2I · I2V · TTS · Editor                        │
├──────────────────────────────────────────────────────────┤
│  ④ 记忆层  memory                                          │
│     角色注册表(一致性) · 叙事记忆(连贯性)                    │
├──────────────────────────────────────────────────────────┤
│     数据契约  models —— ProjectState 全局唯一真相源(黑板)    │
└──────────────────────────────────────────────────────────┘
```

## 3. 核心机制：黑板模式（ProjectState）

整个系统的灵魂。所有智能体**不互相直接调用**，而是统一读写同一份 `ProjectState`：

```go
type ProjectState struct {
    Project    Project                // 创意输入
    Outline    Outline                // 剧本引擎写入
    Characters []Character            // 剧本引擎写入 → 资产管理器回填一致性要素
    Scenes     []Scene
    Shots      []Shot                 // 分镜/音频并发回填产物路径
    Assets     []Asset                // 产物登记
    FinalVideo string                 // 合成器写入
    Status     map[string]NodeStatus  // 各节点状态（断点续跑依据）
}
```

数据单向流动，每个智能体只认契约、不认上游是谁：

```
创意 →[剧本引擎]→ Outline/Characters/Shots
                      ↓
            [资产管理器]→ 回填 Character.{RefImage,Seed,VoiceID}
                      ↓
   ┌──────────────────┴──────────────────┐
[视觉分镜]→ 回填 Shot.{KeyframePath,ClipPath}   [音频合成]→ 回填 Shot.AudioPath
   └──────────────────┬──────────────────┘
                      ↓
            [后期合成]→ 写入 FinalVideo
```

**这是消灭"流程割裂"的根本手段**：流程不靠人工搬运，而靠一份强类型、可持久化的共享契约串起来。

## 4. 总控调度层

### 4.1 DAG 流水线

```
剧本引擎 → 资产管理 → ┬→ 视觉分镜 ┬→ 后期合成
                      └→ 音频合成 ┘
```

分镜与音频都只依赖角色资产就绪，故在 DAG 中并列（无相互依赖）；后期合成依赖二者全部完成。依赖关系声明在 `pipeline.go`。

### 4.2 两级并发模型

| 层级 | 策略 | 原因 |
|------|------|------|
| **节点级** | 按 DAG 拓扑顺序驱动 | 保证可预测、可断点续跑 |
| **镜头级** | goroutine + 信号量并发 | 镜头相互独立，是真正的算力瓶颈所在 |

镜头级并发是 Go 的核心优势所在（`storyboard.go` / `audio_synth.go`），并发度由 `SHOT_CONCURRENCY` 控制。

### 4.3 断点续跑

每个节点完成即落盘 `project.json`。重跑时：
- 节点级：状态为 `DONE` 的节点整体跳过；
- 产物级：已存在且非空的文件跳过生成（`fsx.Exists`）。

视频生成既慢又贵，这是关键工程能力。实测续跑命中缓存 0 秒完成。

## 5. 三大行业痛点的工程化解法

### 5.1 流程割裂 → 黑板模式 + 统一契约

如第 3 节，全流程一份 `ProjectState` 贯穿，无人工搬运、无格式转换断点。

### 5.2 角色一致性差 → 角色注册表

**痛点**：同一角色在不同镜头里长相、画风、声音飘忽。

**解法**（`memory/character_bank.go`）：为每个角色一次性**锁定三要素**，后续所有镜头强制复用：

| 要素 | 作用 | 本地实现 | 真实模型对接 |
|------|------|---------|------------|
| `Seed` | 画风一致 | 由角色 ID 稳定派生，决定配色 | 作为采样种子 |
| `RefImage` | 长相一致 | 角色参考图 | IP-Adapter / InstantID 注入 |
| `VoiceID` | 声音一致 | macOS say 音色名 | 音色克隆 ID |

一致性由架构强制保证，而非每次生成碰运气。

### 5.3 长篇叙事断裂 → 分层递进 + 叙事记忆

**痛点**：长视频里 AI"失忆"，后段与前段脱节。

**解法**：
1. **分层递进生成**（`script_engine.go`）：大纲 → 角色 → 分镜，上层产物作为下层生成的上下文约束，逐级细化；
2. **叙事记忆**（`memory/story_context.go`）：把全局大纲 + 已发生剧情摘要注入每次下层生成的提示词，让生成时始终"记得"故事全局。

## 6. 能力服务层：可插拔设计

智能体只依赖接口，不关心背后实现。换供应商只改 `services/factory.go`：

| 能力 | 接口 | 默认实现（零成本） | 可替换为 |
|------|------|------------------|---------|
| LLM | `LLM` | StubLLM（离线确定性） | DeepSeek / Ollama / GPT（OpenAI 兼容） |
| 文生图 | `T2I` | LocalT2I（SVG→qlmanage→PNG） | FLUX / SDXL / 通义万相 |
| 图生视频 | `I2V` | LocalI2V（ffmpeg zoompan 运镜） | 可灵 / 即梦 / Runway / SVD |
| 语音 | `TTS` | SayTTS（macOS say）/ SilentTTS | CosyVoice / ElevenLabs |
| 剪辑 | `Editor` | FFmpegEditor | — |

**默认配置零 API key 即可产出真实 mp4**，这是对"低成本 + 跑通闭环"的直接回答。

## 7. 本地媒体实现要点

- **关键帧（T2I）**：Homebrew 的 ffmpeg 未编译 `drawtext`（缺 libfreetype），无法直接在画面渲染中文。改用 SVG 排版 → macOS `qlmanage` 渲染为 PNG → ffmpeg `scale+pad` 规整到精确 16:9（qlmanage 按最长边缩放，必须修正宽高比）。
- **运镜（I2V）**：ffmpeg `zoompan` 滤镜模拟推/拉/摇/移/固定，把静态关键帧变成有镜头语言的动态片段。
- **音画对齐**：合成器先探测配音时长，再按该时长重做等长运镜片段，保证每句台词说完再切镜。

## 8. 产物目录结构

```
workspace/{project_id}/
├── project.json          # ProjectState 持久化（断点续跑依据）
├── characters/           # 角色参考图（一致性锁定）
├── shots/                # 各镜头关键帧 + 运镜片段
├── audio/                # 各镜头配音
└── final/
    └── output.mp4        # 最终成片
```
